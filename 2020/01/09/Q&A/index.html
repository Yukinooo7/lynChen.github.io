  2. Briefly explain the differences between training set, validation set and test set for supervised learning.

    • Training set: Learn a model using the training data; give the model the knowledge to classfy the data into different classes.

    • Validation set: is used for model selection

    • Testing set: for final model prediction error; Test the model using unseen test data to assess the model accuracy, usually used at the final process of the artificial neural network.

    • Training set: a set of examples used for learning: to fit the parameters of the classifier In the Multilayer Perceptron (MLP) case, we would use the training set to find the “optimal” weights with the back-prop rule

    • Validation set: a set of examples used to tune the parameters of a classifier In the MLP case, we would use the validation set to find the “optimal” number of hidden units or determine a stopping point for the back-propagation algorithm

    • Test set: a set of examples used only to assess the performance of a fully-trained classifier In the MLP case, we would use the test to estimate the error rate after we have chosen the final model (MLP size and actual weights) After assessing the final model on the test set, YOU MUST NOT tune the model any further

  1. Explain the ‘momentum term’ that is often included in the back-propagation learning algorithm
  • The momentum is to stablize the weights change using a combination of the gradient decreasing term with a fraction of the previous weight change; where t is the index of the current weight change.
  • Momentum term simply makes the following change to the weight update rule, where α is the momentum term:
    • If α=0, this is the same as the regular backpropagation, where the weight update is determined purely by the gradient descent
    • If α=1, the gradient descent is completely ignored, and the update is based on the momentum, previous weight update rule
  1. Out of two of the following, which is better and why? Online, or offline training MLPs with back-propagation of error
  • Explanation:
    on-line learning, single-step learning (MLP, BP) in contrast to batch-learning changes the weights directly after the presention of a single training pattern. Therefore, the subsequent training pattern will “see” an altered network, and will train the weights according to the “new” pattern. It can happen, that the improvement of the preceding learning step is destroyed thereby.
    This, is the reason why single-step learning can generate a residual error, and at the same time can follow a nonstationary distribution of the data from the process.
    batch-learning is changing the weights once, the complete set of data (the batch) has been presented to the network. Therefore, all the data must be available for every learning step.
    Theory tells, that batch-learning is working with the true gradient. Thus, the residual error is often smaller than the single-step residual error.
    But, since one batch-learning step is only performed after a full set of training data is presented, the weight update frequency is rather slow.

  • The outer product W2 of [1,1,1,-1,-1,-1] with itself is a 6*6 matrix
  • The weight matrix M is (1/6)x(W1+W2)

  1. Describe the architecture of the SOM of Kohonen network. What can the network be useful for? The self-organising process can be said to have four major components: Initialization, competition, cooperation and adaptation. Briefly describe how each component works.
  • The architecture of the SOM: two layers of neurons with all inputs connecting to each output.
  • SOM can be used to reduce the dimensionality
  • Initialization: set a grid which is size and structure fixed priori
  • Competition: Find a winner node which is closest to the center
  • CooperationL: The winning neuron locates the center of a topological neighborhood of cooperating neurons, and the cooperative neurons are also allowed to update.
  • Adaptation: Update the weights in relation to the inputs

  1. In what situations you would use Kohonen SOMs over K-means clustering
  • K-means is one of the simplest unsupervised learning algorithms that solve the well-known clustering problem. The procedure follows a simple and easy way to classify a given dataset through a certrain number of clusters and has a low computational cost.
  • The shortcoming of k-means is that the value of K must be determined beforehand. K-means is a greedy algorithm and is hard to attain the optimum clustering results.
  • In K-means the nodes are independent of each other, clusters are formed through centroid and cluster size.
  • Whereas in SOM (Self Organizing Maps), the number of neurons of the output layer has a close relationship with the class number in the input stack. In this, the clusters are formed geometrically.
  • From the performance point of view, the SOMs perform better than K-means algorithm if the number of clusters decreases. K-means is more sensitive to the noise present in the dataset compared to SOM. K-means is more sensitive to the noise present in the dataset compared to SOM.
  • The number of cluster must be specified in advance
  • This method is not suitable for clusters with non-convex shapes
  • This method is sensitive to noise and outlier elements
  • For K-means, its method is not suitable for clusters with non-convex shapes. Also, the K-means method has to set the number of cluster in advance. Finally, K-means is sensitive to noise and outlier elements. As a result, for tasks which has non-convex shapes, cannot set the number of cluster in advance and has noisy environment, SOM performs better than K-means.

  1. Describe the Rectified Linear Unit (ReLU), which is the most commonly applied non-linear activation function in the Convolutional Neural Network (CNN)
    • The forward and backward passes through an ReLU are both just a simple if statement
    • The sigmoid activation requires computing an exponent
    • This advantage is huge when dealing with big networks with many neurons, and can significantly reduce both training and evaluation times.
    • Sigmoid activations are easier to saturate
    • ReLUs only saturate when the input is less than 0
    • For very deep networks, saturation hampers learning, and so ReLUs provide a nice workaround.
  2. Explain the pooling operation in CNN, and compare two different forms of pooling, namely, max pooling and average pooling.
  • The objective of pooling:
    • Transform the joint feature representation into a new, more usable one that preserves important information while discarding irrelevant detail, the crux of the matter being determine what falls in which category

As for MLP, although it can work well with the data with classes, it cannot do the same thing as CNN in image data.
